Visión General

Arquitectura

Smart Forms es un sistema de formularios dinámicos para procesamiento de documentos legales. El usuario completa un formulario web, el sistema genera un PDF relleno y crea una transacción de firma electrónica en OneSpan.

API Backend

El backend expone 5 endpoints. Los de OneSpan requieren el header X-OneSpan-API-Key.

Endpoint Método Descripción Notas
/api/health GET Verifica que el servidor está activo Se llama antes de cada firma
/api/fill-pdf POST Rellena el PDF con los datos del formulario vía Python/pdfrw multipart/form-data: pdf + json + responseType=base64
/api/sign POST Crea una transacción en OneSpan y devuelve el packageId Requiere X-OneSpan-API-Key
/api/getSigningUrl POST Obtiene la URL de firma para redirigir al firmante Requiere X-OneSpan-API-Key
/api/applications POST GET Guarda y recupera borradores de formulario Devuelve URL con ?app={id} para compartir
/api/prefill/:company/:form POST Pre-completa el formulario desde un sistema externo Documentado en la ruta /{form}/docs de la app

Flujo de Firma

Secuencia

Al hacer click en "Firmar", WizardForm ejecuta 3 pasos en secuencia antes de redirigir al usuario a OneSpan.

Generar PDF

Envía la plantilla PDF + los datos del formulario al backend. Python (pdfrw) rellena los campos y devuelve el PDF en base64.

Crear transacción OneSpan

Construye el transactionJson (firmantes + posiciones de firma desde transactionConfig.json) y lo envía al backend, que lo reenvía a OneSpan. Devuelve un packageId.

Obtener URL de firma

Con el packageId, solicita la URL de OneSpan donde el usuario puede firmar el documento.

Descripción de cada archivo

ArchivoObligatorioContenido clave
formConfig.json Sí Define todos los pasos y campos del formulario. Es la fuente de verdad del formulario.
pdfConfig.json Sí title (título de la página) y templatePdf (nombre del archivo PDF).
appConfig.json Sí Opciones de UI. showJsonOnRevision: true muestra el JSON en la pantalla de revisión.
transactionConfig.json Recom. Configuración estática de la transacción OneSpan: metadata y posiciones de campos de firma en el PDF.
brand.json Sí Identidad visual de la empresa. Compartido entre todos los formularios de la misma empresa.
*.pdf Sí Plantilla PDF con campos rellenables. El nombre debe coincidir con pdfConfig.templatePdf.

Anatomía de formConfig

Referencia

Cada campo en formConfig.json puede tener propiedades que controlan su tipo, validación, visibilidad y transformación de datos.

Propiedades disponibles en un campo de formConfig

    Field --> Type["🎨  Tipo de campo\ntext · textarea\ndate · email · tel\ncheckbox (radio si exclusiveGroup)\nbutton-group · info · subtitle · group"]

    Field --> Validation["✅  Validación\nmask: '##-########-#'\n  → MaskedInputField (solo dígitos)\n  → valida longitud exacta\ndigits: 22\n  → valida dígitos sin formato\nrequired: true"]

    Field --> Visibility["👁  Condicionales\nvisibleIf: { field, value }\n  value puede ser string o array\nenabledIf: { field, value }\nhideOnRevision: true"]

    Field --> Transform["🔄  Transformación de datos\nformatter: 'cbuFormatter'\n  → se aplica en onChange y en importar JSON\nsyncTo: 'otroCampo'\n  → copia el valor a otro campo\nderivedFields: [{ name, value: expr }]\n  → evalúa expresión con el valor actual\n  → ej: split de CBU en 22 dígitos\noptionFields: [{ PDFfield: 'val' }]\n  → mapea opción seleccionada a campos PDF"]
